Redis

简介：

  • 问题：
    根据谷歌的一项研究，假如一个网站在3秒钟或更短时间内没有加载成功，会有 53% 的手机用户会离开。
  • 解决方案：
    缓存是让应用程序加速的重要技术之一。存储的信息越接近 CPU，访问速度就越快。从 CPU 缓存中加载数据比从 RAM 中加载要快得多，比从硬盘或网络上加载要快得多得多。
  • 挑战：
    然而，对开发者来说让分布式缓存正确工作是一个巨大挑战。例如，必须谨慎处理本地缓存失效，即替换或删除缓存条目。每次更新或删除存储计算机本地缓存中的信息时，必须更新分布式缓存系统所有计算机内存中的缓存。

Redis 是其中最流行的开源内存数据存储，可用作数据库、缓存或消息代理。

以下会针对一些基础特性及常用功能做一些说明

一：分布式对象

1.RedissonClient :

RedissonClient为顶层接口，通过RedissonClient接口对象操作Redis数据。

每个Redisson对象实例都有有一个与之对应的Redis数据实例。

Redisson支持各种Redis集群模式，本次测试我们使用单机模式：

初始化Redisson实例：

@Configuration
public class RedissonConfig {

    @Bean
    public RedissonClient redissonClient() {
        Config config = new Config();
        config.useSingleServer()
                .setAddress("${address}")
                .setUsername("${username}")
                .setPassword("${password}");
        return Redisson.create(config);
    }

}
  1. RKeys : 所有与Redis key相关的操作都归纳在RKeys这个接口里(请谨慎使用，getKeys()可能会造成redis阻塞)
RKeys keys = redissonClient.getKeys();
keys.getKeys().forEach(System.out::println);
  1. RBucket : 通用对象桶
RBucket<String> bucket = redissonClient.getBucket("bucket");
bucket.set("elem1");
System.out.println(bucket.get());
bucket.trySet("elem2");
bucket.compareAndSet("elem3", "elem4");
bucket.getAndSet("elem5");
System.out.println(bucket.get());

二：分布式集合

Java丰富的集合框架，Redisson中基本都有与之对应的分布式集合对象，使用起来非常方便。

  • Map:

    1. Redisson的分布式Map结构RMap 对象实现了java.util.concurrent.ConcurrentMap和java.util.Map接口，同时还保持了元素的插入顺序。

      RMap<String, String> map = redissonClient.getMap("map");
//设置value，并返回先前的关联value，新key就返回null
String prevValue = map.put("1", "elem1");
//判断hash是否存在，存在就不插入返回先前value，不存在就插入并返回null
String currValue = map.putIfAbsent("2", "elem2");
//移除Key。返回先前value
String removedValue = map.remove("1");
//比put快，不返回以前的值，返回boolean。 true代表key是哈希中的新密钥，并且已设置值。false代表密钥已经存在于哈希中并且值已更新
boolean putResult = map.fastPut("3", "elem3");
//异步 设置值,并返回先前的关联值，新key就返回null
RFuture<String> putAsyncFuture = map.putAsync("4", "elem4");
//异步 true如果key是哈希中的新密钥，并且已设置值。 false如果密钥已经存在于哈希中并且值已更新
RFuture<Boolean> fastPutAsyncFuture = map.fastPutAsync("5", "elem5");

    2. 淘汰机制：

      目前的Redis自身并不支持哈希（Hash）当中的元素淘汰，因此所有过期元素都是通过org.redisson.EvictionScheduler实例来实现定期清理的。为了保证资源的有效利用，每次运行最多清理100个过期元素。任务的启动时间将根据上次实际清理数量自动调整，间隔时间趋于1秒到2小时之间。比如该次清理时删除了100条元素，那么下次执行清理的时间将在1秒以后（最小间隔时间）。一旦该次清理数量少于上次清理数量，时间间隔将增加1.5倍。

      RMapCache<String, String> cacheMap = redissonClient.getMapCache("anyMap");
//有效时间 ttl = 10分钟
cacheMap.put("key1", "key1", 10, TimeUnit.MINUTES);
//有效时间 ttl = 10分钟, 最长闲置时间 maxIdleTime = 10秒钟
cacheMap.put("key1", "key1", 10, TimeUnit.MINUTES, 10, TimeUnit.SECONDS);
//有效时间 = 3 秒钟
cacheMap.putIfAbsent("key2", "key2", 3, TimeUnit.SECONDS);
//有效时间 ttl = 40秒钟, 最长闲置时间 maxIdleTime = 10秒钟
cacheMap.putIfAbsent("key2", "key2", 40, TimeUnit.SECONDS, 10, TimeUnit.SECONDS);

  • Set:

    1. Redisson的分布式Set结构的RSet对象实现了java.util.Set接口。通过元素的相互状态比较保证了每个元素的唯一性。

      RSet<String> set = redissonClient.getSet("set");
set.add("elem1");
set.add("elem1");
set.remove("elem1");
Assert.isTrue(set.size() == 0, "uniqueness test failed");

    2. 淘汰机制：

      同RMap的淘汰机制。

      RSetCache<String> cacheSet = redissonClient.getSetCache("cacheSet");
// ttl = 10 seconds
cacheSet.add("elem1", 10, TimeUnit.SECONDS);

  • SortedSet:

    Redisson的分布式RSortedSet对象实现了java.util.SortedSet接口，在保证元素唯一性的前提下，通过比较器(Compaarator)接口实现了对元素的排序。

    RSortedSet<Integer> sortedSet = redissonClient.getSortedSet("sortedSet");
// 配置元素比较器
sortedSet.trySetComparator(new MyComparator()); 
sortedSet.add(3);
sortedSet.add(1);
sortedSet.add(2);
sortedSet.removeAsync(0);
sortedSet.addAsync(5);

  • List:

    Redisson分布式列表RList 对象在实现了java.util.List接口的同时，确保了元素插入时的顺序。

    RList<String> list = redissonClient.getList("list");
list.add("elem");
String result = list.get(0);
Assert.isTrue(result.equals("elem"), "RList add failed");
list.remove("elem");
Assert.isTrue(list.size() == 0, "RList remove test failed");

  • More:

    更多 (ScoredSortedSet、LexSortedSet 、Queue、Deque、Blocking Queue 、Bounded Blocking Queue 、Blocking Deque 、Blocking Fair Queue 、Delyed Queue 、Priority Queue) 请参考：

三：分布式锁

Jedis和Lettuce关于分布式锁都要自己手动去实现，而Redisson对此有完整的封装。

Redisson的分布式可重入锁RLock对象实现了java.util.concurrent.locks.Lock接口，并且还支持自动过期解锁。

简单实现分布式锁：

RLock lock = redissonClient.getLock("lockKey");
// 最常见的使用方法
lock.lock();
// 支持过期解锁功能
// 10秒钟以后自动解锁
// 无需调用unlock方法手动解锁
lock.lock(10, TimeUnit.SECONDS);
// 尝试加锁，最多等待100秒，上锁以后10秒自动解锁
boolean res = lock.tryLock(100, 10, TimeUnit.SECONDS);

同时Redisson可以完美实现fairLock、RedLock、ReadWriteLock等

四：高级特性

  1. Geospatial Bucket

    Redisson的分布式RGeo对象是一种专门用来储存与地理位置有关的对象桶。

    RGeo<String> geo = redissonClient.getGeo("test");
geo.add(new GeoEntry(13.361389, 38.115556, "Palermo"),
        new GeoEntry(15.087269, 37.502669, "Catania"));
geo.addAsync(37.618423, 55.751244, "Moscow");

Double distance = geo.dist("Palermo", "Catania", GeoUnit.METERS);
geo.hashAsync("Palermo", "Catania");
Map<String, GeoPosition> positions = geo.pos("test2", "Palermo", "test3", "Catania", "test1");

  2. Bloom Filter

    Redisson利用RBloomFilter实现了Java分布式布隆过滤器，这在分布式环境中大数据量去重有显著的效果。

    RBloomFilter<String> bloomFilter = redissonClient.getBloomFilter("sample");
// 初始化布隆过滤器，预计统计元素数量为55000000，期望误差率为0.03
bloomFilter.tryInit(55000000L, 0.03);
bloomFilter.add("field1Value");
bloomFilter.add("field2Value");
bloomFilter.contains("field3Value");

  3. RRateLimiter

    Redisson利用RRateLimiter实现了分布式布限流器，这在分布式环境中在网关层对于QPS限制等方面特别易于使用。

    RRateLimiter rateLimiter = redissonClient.getRateLimiter("myRateLimiter");
//最大流速 = 每1秒钟产生100个令牌
rateLimiter.trySetRate(RateType.OVERALL, 1, 100, RateIntervalUnit.SECONDS);
//阻塞直到一个可用。
//获取给定数量的许可证，如果它们可用并立即返回，减少给定数量的可用许可证数量。
//返回是否获取成功
boolean succeed = rateLimiter.tryAcquire(1);

  4. 更多高级特性请参考wiki

五：Quick Start

1. 添加redisson-spring-boot-starter依赖

dependencies {
    implementation 'org.redisson:redisson-spring-boot-starter:3.16.0'
}

2. 配置redis连接

spring:
  redis:
    host: ${host}
    port: 6379
    username: ${usrname}
    password: ${password}

更过配置参数参考 org.redisson.config.Config

3. 使用

@Autowired
private RedissonClient redissonClient;

@Test
public void testRedis() {
    RList<String> list = redissonClient.getList("list");
    list.add("elem1");
    list.add("elem2");
    list.forEach(System.out::println);
    list.clear();
}
